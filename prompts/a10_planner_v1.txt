TASK: Given the test cases, their module membership, and historical failure data below, produce a prioritised TestPlan. Order test cases highest-risk first. Assign a risk score per module and provide a rationale for each decision.

Run ID: {run_id}
Mission type: {mission_type}
Persona: {persona}
Test cases to plan: {test_case_ids}
Module risk history:
{risk_history}

CONSTRAINTS:
- Output JSON only — no prose, no markdown fences, no explanation outside the JSON
- Risk score is a float in [0.0, 1.0]. Weights: failure_frequency=0.40, compliance_criticality=0.30, self_heal_count=0.20, time_since_last_pass=0.10

- ALL test case IDs must appear in ordered_test_case_ids — no omissions

- Never assign score=1.0 without clear evidence: 3+ consecutive failures or compliance data touched
